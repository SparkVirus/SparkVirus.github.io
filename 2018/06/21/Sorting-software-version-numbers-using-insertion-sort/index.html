Sorting software versions using insertion sort

Created at 21-06-2018 Updated at 25-06-2018 Category KABU ihub Tag insertionsort / Java / C++

In software versioning, the semantic versioning scheme i.e MAJOR.MINOR.PATCH, is the most commonly used to assign versions to every software released. In this scheme, when backward-incompatible API changes are made the MAJOR version is incremented, when backword-compatible API changes are made the MINOR version is incremented and when backward-compatible bug fixes are made only the PATCH version is incremented.

Task

Provided with a string array of versions as the input, write a program that would print to the versions in an ascending order i.e From earliest release to the latest. A version can contain the MAJOR version only, MAJOR and MINOR versions only or contain all the parts of the semantic versioning system.

Example

Sample input:

["1.1.2", "1.0", "1.3.3", "1.0.12", "1.0.2", "2.0", "15.45", "15.09", "0.0", "1", "0", "0.0", "2.0.1", "2.1.2", "0.1.2", "0.1"]

Solution

Before we begin sorting though the versions we need to find a way to compare the versions, since each version is actually a string. So how do we figure out that a given version was released earlier than another?

Let us first consider only two versions. Let the first version be v1 = M1.m1.p1 and the second version be v2 = M2.m2.p2. From this we can make the following deductions:

  • v1 will always be greater v2 if M1 > M2 and vice versa.(1)
  • If M1 and M2 are equal, v1 will always be greater than v2 if m1 > m2 and vice versa.(2)
  • If M1 and M2,and m1 and m2 are equal, v1 is greater than v2 if p1 > p2 and vice versa.(3)

Now suppose we have a version in which we have been provided with MAJOR version only or the MAJOR version and the MINOR versions only. So now we let v1 = M1 and v2 = M2.m2. From this we can deduce that:

  • v1 can be represented as v1 = M1.0.0 while v2 can be represented as v2 = M2.m2.0 and therefore, our first two deductions apply.(4)
  • If M1 and M2 are equal and m2 = 0 then we will pick v1 to come before v2. (5)

Now after we have made the above deductions we can put them into code. So to solve our problem we will use the insertion sort algorithm to sort the versions. You can read about Insertion Sort algorithm to gain some understanding then continue afterwards. I will write a java version of the code and an equivalent in C++.

To do any comparisons between the version we will need a method that will split each version string and compare the parts. The method will take two parameters; the first parameter as the first version being compared to the second parameter representing the second version.
